in ▸ user actions; the adapter's SSE stream; REST responses.

Adapter API FastAPI the one backend

The single backend. Routes under /api/* (runs, sessions, projects, settings, search, models, stats). Owns supervision/approval policy and the SSE response for …/events.

in ▸ HTTP requests from the UI.
out ◂ JSON responses + an SSE event stream; drives the prover via the Bridge.
Bridge prover seam · SSE map the in-process seam

Composes project context into messages, builds LeaConfig, calls run_events(cfg, …) in-process, maps the prover's typed events onto SSE, and runs the per-tool approval gate.

in ▸ session/project from DB+git; prover typed events.
out ◂ SSE events to the UI; git commits + SQLite index rows.
Agent loop 6 tools vendored prover

The proving loop: write → compile → read errors → edit, repeat. Tools: read_file · write_file · edit_file · lean_check · bash · search_mathlib. Streams model tokens from the providers.

in ▸ composed messages + config; tool results; model tokens.
out ◂ tool calls; file writes; the typed event stream.
LSP daemon · SafeVerify · Mathlib search prover services

LSP daemon — warm lean_check (~0.2s edits). SafeVerify — kernel replay + axiom whitelist (the honest "verified" verdict; catches structural cheats). Mathlib search — lemma lookup (budget 20/problem).

in ▸ a .lean file path / a search query.
out ◂ compile diagnostics · kernel-replay verdict · lemma matches.
Lean + Mathlib · Model catalog local resources

Lean + Mathlib — the Lake workspace (Lean v4.29, pinned Mathlib); proofs at proofs/. Model catalog — models.json, the provider/model list read by both the UI and the adapter.

Git store · SQLite index persistence · source of truth

Git owns proof content (commit on every FileChanged). SQLite is a rebuildable index (sessions · runs · code_steps · messages · projects · usage) — never proof bytes. Session status is derived from the latest code-step verdict, so it can't drift.

Model providers external · cloud

Gemini · Anthropic · OpenAI, reached over HTTPS. Keys come from the root .env / Settings (redacted, presence-only to the client).
