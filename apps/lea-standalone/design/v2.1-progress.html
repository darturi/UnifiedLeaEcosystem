Known gaps · deferred (not blocking)

SafeVerify can't audit a project proof that imports a sibling lemma. A project submission that does import Lea.<Project>.Foo fails SafeVerify with unknown module prefix 'Lea': safe_verify replays oleans inside its own Lake project (third_party/SafeVerify), whose search path has only its packages + Mathlib — not the main workspace's compiled Lea.* oleans (workspace/.lake/build/lib/lean). Loose proofs (single file, Mathlib-only imports) are unaffected; chaining proofs surfaced it (Q2/F3 testing). Fix direction: when the submission imports project modules, add the workspace build lib to safe_verify's LEAN_PATH (and ensure the sibling olean is built) in prover/lea/safeverify.py → verify_proof (the lake exe safe_verify invocation, cwd=SAFE_VERIFY_DIR).
